# i18n (lot 0)

Multilingual support for doxyde sites. Content is authored in **English** (the
canonical/source language); every other language is **machine-translated on
demand** by an external translation microservice and cached per-site in SQLite.

## How it works

- **Serving language** is resolved per request: cookie `lang` → `Accept-Language`
  → the site's `default_lang`, clamped to the site's enabled set
  (`locale_middleware.rs`). `<html lang/dir>` is set accordingly (RTL supported).
- **Language switch** uses page-scoped dot-actions `/<page>/.fr`, `/<page>/.en`,
  … They render the page in the target language and respond **200 + `Set-Cookie:
  lang=…` (never a redirect)**, so the URL stays a deterministic, indexable
  per-language canonical. `hreflang` alternates point at these URLs; the bare URL
  is `x-default`.
- **Two layers**: UI labels (`ui_catalog.rs` catalog → `ui_labels::resolve_labels`
  → injected as the Tera `labels` map) and editorial content (page title/
  description + component text fields, translated in place by
  `content_translate.rs` before rendering — no template changes).
- **Translation policy**: the canonical `/.fr` `/.en` URLs translate
  **bounded-synchronously** (short timeout, batched) so crawlers get translated
  content on the first hit; the cookie-served bare URL is **deferred** (serve
  source now, enqueue a background job, translation appears on a later load). If
  the synchronous attempt times out, it serves source AND enqueues a deferred job,
  so the canonical URL **self-warms**: the next crawl is served the cached
  translation. Both cache by `sha256(source)` in `translation_cache`, so editing
  the English source naturally invalidates its translation. Manual overrides
  (`is_manual_override=1`) are never clobbered by the worker.
- **Cold-hit latency**: a page translates its content and its nav/breadcrumb
  titles as two bounded batches, so a cold canonical hit can wait up to ~2×
  `DOXYDE_I18N_SYNC_TIMEOUT_MS` before falling back to source (then warms in the
  background). Tune the timeout to taste — correctness no longer depends on it.

## Migrations on existing site DBs

New migrations now apply to **existing** site databases on first access, not only
to freshly created ones (`db_router` calls `doxyde_db::run_migrations` on connect;
idempotent via `_sqlx_migrations`). Before this, a migration added after a site's
creation was never applied to that site — so any new per-site schema (including
this i18n migration) would silently never reach live sites.

Per-site config lives in the `i18n_config` (singleton) and `i18n_enabled_lang`
tables (migration `021_i18n.sql`); default enabled set is `{en, fr}`.

## Translation microservice (shared, on atlantic)

doxyde does not call OpenRouter directly — it speaks a small TCP protocol to the
reusable `i18n-service` (from `~/yatoo.travel/i18n-service`). Deploy **one shared
instance** bound to `127.0.0.1:4003` (never internet-exposed).

```sh
# Build & place the service (from the yatoo i18n-service sources)
ssh root@atlantic 'mkdir -p /opt/doxyde-i18n'
# rsync the i18n-service crate to /opt/doxyde-i18n, then:
ssh root@atlantic 'cd /opt/doxyde-i18n && cargo build --release'

# /opt/doxyde-i18n/.env  (bind localhost only; key from ~/aios/.env convention)
#   HOST=127.0.0.1
#   PORT=4003
#   OPENROUTER_API_KEY=<DOXYDE_OPENROUTER_API_KEY from ~/aios/.env>
#   CACHE_DIR=/opt/doxyde-i18n/cache
#   (optional) OPENROUTER_MODELS=...

# Install the unit and start it
scp deploy/systemd/doxyde-i18n.service root@atlantic:/etc/systemd/system/
ssh root@atlantic 'systemctl daemon-reload && systemctl enable --now doxyde-i18n'
```

## doxyde configuration

Set in doxyde's EnvironmentFile (defaults shown):

```
DOXYDE_I18N_SERVICE_ADDR=127.0.0.1:4003
DOXYDE_TRANSLATION_WORKERS=4
DOXYDE_I18N_SYNC_TIMEOUT_MS=3000
```

Rebuild & restart (migration 021 applies automatically to each per-site DB on
startup). Start `doxyde-i18n` **before** doxyde so the first translations succeed;
doxyde tolerates the service being down regardless (serves source, retries later).

```sh
ssh root@atlantic "cd /opt/doxyde && git pull && cargo build --release && systemctl restart doxyde doxyde-sse"
```

## Local verification

```sh
# 1. Run the translation service locally
cd ~/yatoo.travel/i18n-service && OPENROUTER_API_KEY=… PORT=4003 cargo run --release
# 2. Run doxyde pointed at it
cd ~/doxyde && DOXYDE_I18N_SERVICE_ADDR=127.0.0.1:4003 cargo run -p doxyde-web
```

Checks:
- `GET /` → `<html lang="en" dir="ltr">`, English, no service calls.
- cookie `lang=fr`, `GET /` → 200 source immediately; reload → French from cache.
- `GET /<page>/.fr` → **200 + `Set-Cookie: lang=fr`** (not 301/302), French on the
  first hit, `<link rel="canonical" …/.fr>`, hreflang fr/en/x-default present.
- `Accept-Language: fr` (no cookie) → French; a conflicting cookie wins.
- Enable `ar`, hit `/<page>/.ar` → `<html lang="ar" dir="rtl">`.
- Stop the service → French page still 200 with source text (no hang/500).

## Known scope limits (lot 0)

- `ui_label_cache` is created for a future admin label-override UI but not yet
  written by the resolution path (labels resolve through `translation_cache`).
- `blog_summary` lists embed child-page titles built before translation, so those
  list entries stay in the source language (page body, current-page title/desc,
  components, top nav, breadcrumb and hierarchical nav ARE translated).
